# 그룹화는 Tag(N:M, 옵셔널)로만 한다 — Project/List/Folder 컨테이너 없음

To-do 도구는 거의 모두 `Project`(또는 `List`, `Folder`) 컨테이너를 1급으로 둬서 **Task**가 정확히 하나의 컨테이너에 속하도록 강제한다. 이 강제는 1인 운영에서 거의 항상 과한 의식이 되고("이 Task는 어디에 속하지?"라는 질문 자체가 마찰), 컨테이너에 들어가지 않는 산발적 **Task**를 어색하게 만든다.

**결정:** **Tag** 하나만 1급 그룹화 수단으로 둔다. **Task** 1건은 0..N개의 **Tag**를 가지고, **Tag** 1개는 0..N개의 **Task**에 붙는다(다대다, 옵셔널). `Project`/`List`/`Folder`/`Bucket`/`Area`는 _Avoid_.

**되돌리는 조건:** 30개 이상 **Task**가 모인 강한 묶음이 자연스럽게 형성되고, **Tag** 필터만으로는 진행률·완료율 같은 묶음 수준 통계를 보고 싶어졌을 때.
