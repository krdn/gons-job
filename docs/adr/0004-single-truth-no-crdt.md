# 단일 truth + 단순 복제 — 충돌 해소(CRDT/Vector clock 등)는 도메인 밖

멀티 디바이스 동기화는 1인 도구에서도 흔한 기대지만, 충돌 해소 모델(CRDT, vector clock, causal order, tombstone)을 도메인의 일부로 들이는 순간 **Task**의 모든 변경이 "어느 디바이스의 어느 시점 버전인가"라는 메타데이터를 끌고 다녀야 한다. 1인 운영에서 동시 편집 충돌이 실제로 발생할 빈도는 매우 낮고, 발생하더라도 last-write-wins로 충분하다.

**결정:** 한 디바이스(또는 로컬 git/파일 동기화 폴더)가 단일 truth다. 다른 디바이스는 단순 복제(파일 동기화, git pull, 또는 LWW)로 접근한다. 도메인 모델은 "현재 상태"만 표현하고, 충돌 해소·동기화 메타데이터는 도메인 밖이다. `Sync`/`Replication`/`Merge`/`Conflict`/`CRDT`/`VectorClock`/`CausalOrder`/`Tombstone` 어휘는 _Avoid_.

**되돌리는 조건:** 두 디바이스에서 동시 오프라인 편집 후 충돌로 데이터를 잃는 사건이 실제로 발생했을 때, 또는 다른 사람과의 공유가 도메인 요구로 들어왔을 때(이 경우 ADR-0001~0004 전반의 1인 가정도 함께 재검토).
